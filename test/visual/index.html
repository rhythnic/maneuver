Maneuver

  • github
  • npm
  • nib

Maneuver is a collection of stylus mixins to facilitate creating interactive css elements. Some of the mixins help with creating the layout, most will require coordinating HTML and JavaScript along with the mixins, and all of them will help you to create the effect of movement and transition on the page.

Offcanvas

The offcanvas mixin allows you to create a menu that hides outside of it's parent element, and opens/closes by adding/removing a class on the parent element.
The menu on the left side of this page is an example of the offcanvas mixin.

HTML


<div class="my-wrap">
  <div class="menu"></div>
  <div class="content">
    <button>Toggle menu</button>
  </div>
</div>

Stylus


.my-wrap
    offcanvas()

JavaScript


$('button').on('click', function (e) {
  $('.my-wrap').toggleClass('open');
})

Specification

There are 2 mixins for creating an offcanvas menu: offcanvas() and offcanvasMulti(). Multi allows you to create 1-4 offcanvas menus. Both have similar parameters. The paramters all have default values unless specified.

offcanvas()

parameters and defaults:
side: 'left'
menu position. Options are 'top', 'right', 'bottom', or 'left'
size: 15em
width of left/right menus, height of top/bottom menus
share: 40em
min screen width at which menu will share the container with content when menu is open
show: 80em
min screen width at which menu can't be hidden.
push: 0
only applies when share or show don't apply. set to 0 to have menu go on top of the content. set to 1 to have the menu push the content.
menu: '.menu'
css selector for the menu container
content: '.content'
css selector for the content container
open: 'open'
class that gets toggled on the parent container for open/close
transition: all 0.3s ease
css rule for transitions

Things to note:

  • To disable share or show, set it to 0.
  • To enable share or show at any screen size, set it to 0em.
  • The offcanvas menu and content adopt the height of the parent element. For a full page offcanvas menu, you'll need to specify html, body, and your parent element as height 100%.
  • In the previous example, all of the default selectors were used, including .menu-left, .content, and .open. These can all be specified.
  • Offcanvas doesn't have any css rules for the toggle button, just the menu and content containers. The content area has position: relative, so you can set the button as position: absolute and use top, right, bottom, left css properties.

offcanvasMulti()

OffcanvasMulti has basically the same parameters, but there is no 'side' parameter. Instead, you should declare settings for all 4 sides. You can specify all 4 parameters similar to how you would write css shorthand, "(top right bottom left)". The default of offcanvasMulti is a left and right menu, both with the same defaults as offcanvas(). Be careful in calling offcanvasMulti. It's better to specify more than is necessary than not enough.

parameters and defaults:
size
(0 15em 0 15em)
share
(0 40em 0 40em)
show
(0 80em 0 80em)
push
(0 0 0 0)
menu
('.menu-top' '.menu-right' '.menu-bottom' '.menu-left')
content
'.content'
open
('open-top' 'open-right' 'open-bottom' 'open-left')
transition
all 0.3s ease

Examples

offcanvas(side: 'top', size: 8em, share: 0, show: 0, menu: 'nav', content: '.main')
offcanvas(side: 'right', size: 12em, share: 0em, show: 0)
offcanvasMulti(size: (10em 0 5em 0), share: (0 0 0 0), show: (0 0 0 0))

Toggle

Toggle styles a checkbox as an on/off switch. You can customize most aspects of it. You can put a label on the switch, you can change the on/off text, and you can specify all the colors in both the on and off states.

HTML


<div class="my-toggle">
  <input type="checkbox" id="my-setting">
  <label for="my-setting"><span>Heater</span></label>
</div>

Stylus


.my-toggle
  toggle()

Specification

toggle()

parameters and defaults:
switch: (#b2b3a1 #000)
(background-color [text-color]), colors for the moving part of the toggle
switch_on: (#b2b3a1 #000)
(background-color text-color), optional alternate colors for the switch in on position
off: (#bb9297 #000 'OFF')
(background-color text-color [content]), colors and content for the 'off' part of the toggle
on: (#29c910 [inherit 'ON'])
(background-color text-color [content]), colors and content for the 'on' part of the toggle
side_width: (total_width / 2.5)
width of the on/off parts
shadow: 1
inset shadow for on/off parts, set shadow to 0 to disable

Things to note:

  • Checkbox must have an id that matches the 'for' attribute of label. This is because checkbox is hidden and the switch is actually the label. Setting the 'for' attribute allows us to toggle the checkbox when the label is clicked.
  • The span element inside of the label is necessary. This allows us to vertically center the text.

Examples

toggle(switch: (#6C7F7E, #000), switch_on: (lighten(#6C7F7E, 50%) #fff), off: (#DC3522 #000 'NO'), on: (#D9CB9E #fff 'YES'))

.my-toggle
  width 15em
  height 5em
  font-size: 2em
  toggle(side_width: 3em)
